# PATCH NOTES — piTrainer_0_9_14 Session Working Folder and Top Bar Removal

## Request summary

Patch the v9 app so that when a session path/session is selected, all default output locations follow that session folder, including:

- preprocess save/settings path
- trained model save path
- export output path
- validation / TFLite / deploy model search defaults through the shared export folder state

Also remove the page-level top bar shown above the main workspace for now.

## Cause / root cause

The app already tried to sync some output paths after a session was loaded, but the behavior was still too late and incomplete for the user workflow:

- output folders were synced mainly after `Load Selected`, not immediately when the session selection changed;
- selecting a direct session folder as the records root could still leave output folders on the older trainer output folder until a session row was loaded;
- preprocess settings path followed the shared output folder but did not reload session-local preprocess settings when that folder changed;
- the page banner/top bar was still globally generated by `DockPage.set_workspace_widget()`, so every page showed the large workflow header.

## Files changed

- `piTrainer/piTrainer/main_window.py`
  - Adds direct-session-folder detection.
  - Extends working-folder sync so it can run on selection changes as well as after loading.
  - Syncs shared export output, trained-model save folder, preprocess path, and validation/export-validation refreshes from the same working folder.
- `piTrainer/piTrainer/pages/data_page.py`
  - Passes a session-selection-change callback into the Session Source panel.
- `piTrainer/piTrainer/pages/data_page_sessions.py`
  - Syncs output paths when sessions are selected, not only after loading.
  - Auto-selects the `.` session when the chosen records root is itself a direct session folder.
  - Re-syncs the working folder after refreshing sessions.
- `piTrainer/piTrainer/panels/data/session_source_panel.py`
  - Adds a selection-changed callback for checkbox changes, Select All, Clear, and programmatic selection updates.
  - Suppresses repeated callbacks while rebuilding the session list.
- `piTrainer/piTrainer/pages/preprocess_page.py`
  - Updates preprocess save/settings path whenever the working folder changes.
  - Loads session-local preprocess settings if they already exist in the selected session folder.
- `piTrainer/piTrainer/pages/dock_page.py`
  - Disables the page-level top banner globally while keeping the same page API for future restore.
- `piTrainer/piTrainer/version.py`
  - Updates visible version to `0.9.14 / piTrainer_0_9_14`.
- `piTrainer/PATCH_NOTES/PATCH_NOTES_piTrainer_0_9_14.md`
  - Adds this patch note.

## Exact behavior changed

- Selecting/ticking a session in Data > Load now immediately sets the app working folder to that selected session folder.
- Loading a selected session still sets the working folder to the loaded session folder.
- Selecting multiple sessions sets the shared working folder to the records root, because there is no single session folder.
- Choosing a records root that is itself a direct session folder now uses that folder as the working folder and auto-selects the direct `.` session row.
- Train > model save folder now follows the selected/loaded session folder.
- Export > output folder now follows the selected/loaded session folder.
- Preprocess save/settings path now follows the selected/loaded session folder.
- Validate, TFLite Check, and Deploy continue to use the shared export folder state, so their latest-model search defaults also follow the selected/loaded session folder.
- The large page-level top bar/banner is removed across the pages. The main splitter workspace now starts directly below the workflow tab strip.

## Behavior intentionally preserved

This patch does not change:

- Data Workflow tab order: `1 Load`, `2 Hide & Recover`, `3 Filter`, `4 Review`, `5 Deploy`.
- Deploy output columns, diff sorting, overlay preview, or Apply AI to Selected.
- Up/Down next-frame navigation fixes.
- Preprocess done banner inside the Preprocess output panel.
- Hide/Recover and hidden permanent-delete behavior.
- The `0.9.11` Data page mixin split.
- The `0.9.12` Preview/Overlay refactor.
- The `0.9.13` Validation service refactor.
