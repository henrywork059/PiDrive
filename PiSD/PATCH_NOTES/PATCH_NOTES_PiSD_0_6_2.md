# PiSD 0.6.2 Patch Notes

## Request summary
Improve the Manual Drive and AI Mode visualised-path overlay when turning, because the v0.6.1 filled corridor still did not look enough like the actual path the car could take during curved movement.

## Cause / root cause
The v0.6.1 overlay improved the presentation by adding a filled road corridor, but the turn centreline was still generated mostly as a screen-space shaped curve. Straight driving looked acceptable because the trapezium is simple, but turning exposed the weakness: the curve could read like a decorative bow rather than a physically plausible vehicle path. The user also highlighted the key implementation risk: road edges should be offset from the local path tangent, not from a fixed screen or X-axis offset after the fact.

## Files changed
- `pisd/__init__.py`
- `pisd/web/static/js/manual_drive.js`
- `pisd/web/static/js/ai_mode.js`
- `PATCH_NOTES/PATCH_NOTES_PiSD_0_6_2.md`

## Exact behaviour changed
- Manual Drive overlay now predicts the centre path with a small kinematic bicycle-style ground-plane trajectory instead of only shaping a 2D screen curve.
- AI Mode overlay now uses the same kinematic sampled path method, keeping AI preview/drive visualisation consistent with Manual Drive.
- The road-edge corridor is now generated by offsetting each sampled centre point using the local tangent normal before projection into the SVG viewbox.
- Turning paths now curl more like a swept future route: tight turns shorten forward reach naturally, bend out to the side more clearly, and keep the filled road area attached to the predicted trajectory.
- Straight driving still produces a perspective trapezium.
- The accepted v6 direction convention is preserved: negative/left steering still shifts the visible future path toward the right side of the camera frame, and positive/right steering shifts it toward the left side.
- Reverse-motion visualisation remains hidden.
- Existing overlay calibration settings are preserved; no runtime config file is reset.

## Verification not performed
- Full Flask route/API tests were not run because Flask is not installed in this container.
- Real Raspberry Pi camera/motor hardware tests were not run in this container.
- Real camera calibration, lens distortion correction, and physical wheelbase/camera extrinsic calibration were not performed. This patch improves the visual model but is still not a calibrated IPM/homography projection.

## Known limits / next steps
- The overlay still uses a visual pseudo-projection inside the browser SVG, not a real camera matrix with distortion coefficients.
- If the physical car video still shows mismatch near the edge of the OV5647 frame during hard turns, the next improvement should add real camera calibration values and a ground-plane homography/IPM path projection.
- Shared overlay logic remains duplicated between `manual_drive.js` and `ai_mode.js`; a future cleanup patch could centralise the geometry helper after the behaviour is accepted on hardware.
