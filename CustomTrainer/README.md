# CustomTrainer 0_1_1

CustomTrainer is now a **PySide6 desktop trainer app** for object-detection workflows, refreshed to follow the same general desktop UI language as **piTrainer** while keeping the CustomTrainer feature set focused on YOLO-style datasets and Raspberry Pi deployment.

## What changed in 0_1_1

- Replaced the previous `tkinter` main UI layer with a **PySide6** desktop shell.
- Added a **piTrainer-style dark Fusion theme** with blue-accent tabs, group panels, status bar, and docked log console.
- Rebuilt the workflow pages in PySide6 for:
  - Dataset setup and scanning
  - Annotation editing
  - Training launch
  - Validation / prediction launch
  - Export launch
  - Pi deployment bundle creation
- Kept the existing service-layer logic for dataset scanning, Ultralytics command building, and Pi bundle generation so the patch stays compatible with the current project structure.

## Main features

- Desktop GUI built with **PySide6**
- Modular pages under `custom_trainer/ui/pages/`
- Docked log console similar to piTrainer’s desktop workflow
- Dataset scanning and `dataset.yaml` creation
- Basic YOLO box annotation editing
- Ultralytics YOLO train / val / predict / export launch
- Pi-friendly TFLite export defaults
- Pi deployment bundle builder with runtime scripts

## Notes

- This patch changes the **UI framework** to PySide6, so `PySide6` is now included in `requirements.txt`.
- The object-detection workflow remains PC-train / Pi-run.
- The Pi runtime bundle contents are still generated by the existing deployment service and bundled runtime assets.
